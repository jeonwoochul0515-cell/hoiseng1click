## 1. 현재 상황 및 문제점

### CODEF PFX 인증 성공
- `certType: "0"` + `certFile`(PFX Base64) 조합으로 CODEF 인증 성공
- connectedId 발급 확인 (`1g0zl3yEQZEbQwGjMFAk8I`)

### 핵심 문제: 공동인증서 타기관 미등록
- 18개 기관 중 **1개(국민은행)만 성공**, 나머지 `CF-12805: 미등록 인증서`
- 공동인증서는 각 금융기관에 **개별 등록**이 필요 — 한번에 일괄 등록 방법 없음
- 의뢰인이 사용하는 인증서가 국민은행에만 등록되어 있었음

### 개인회생 의뢰인의 현실
- 인증서 자체가 없거나 만료된 경우가 **대부분**
- 연체로 인터넷뱅킹 이용 제한
- 계좌 압류/동결 상태에서 인증서 재발급 불가
- 5~10개 이상 기관에 채무 분산 — 각 기관별 인증서 등록은 **비현실적**
- 의뢰인에게 "전 금융기관 인증서 등록"을 요구하는 것은 실무적으로 불가능

## 3. CODEF 활용 최적화 방안

### CODEF에서 Credit4U 스크래핑 가능

- CODEF 개발가이드에 `/products/insurance/each/credit4u/register` 경로 확인
- CODEF가 Credit4U 데이터를 스크래핑 방식으로 제공할 수 있음
- **1회 호출로 전체 금융기관 채무 목록 확보 가능** (개인회생 핵심)
- 데모버전에서 사용 가능 여부는 CODEF에 확인 필요

### 간편인증 활용 범위

| loginType | 인증 방식 | 대상 |
|-----------|----------|------|
| 0 | 공동인증서 (PFX) | 전 기관 (등록된 곳만) |
| 1 | ID/PW | 홈택스, 정부24, 건보공단 |
| 5 | 간편인증 (카카오/PASS) | 주요 은행/카드사 (기관별 상이) |

- 간편인증 loginTypeLevel: 1=카카오, 5=PASS, 6=네이버, 8=토스
- **공공기관(홈택스 등)은 ID/PW로 인증서 없이 조회 가능**
- 간편인증은 의뢰인이 직접 모바일 앱에서 인증 필요

### CODEF + Credit4U 하이브리드 전략 (권장)

```
[1단계] CODEF → Credit4U API
    → 전체 금융기관 채무 목록 일괄 확보 (누락 방지)

[2단계] CODEF → 개별 은행/카드사 (선별적)
    → Credit4U에서 확인된 기관만 상세 조회
    → 간편인증 또는 ID/PW 사용

[3단계] CODEF → 공공기관 (ID/PW)
    → 소득금액증명원, 건강보험, 국민연금
    → 인증서 불필요
```

### 공동인증서 서비스 (50만원/월) 분석

- CODEF가 제공하는 인증서 팝업 UI + 릴레이 서버 + 보관 서비스
- PFX 파일 직접 취급의 보안 리스크 해소
- 월 50만원 = 의뢰인 10명 기준 건당 5만원 추가
- **초기에는 불필요 — 간편인증 + ID/PW로 시작 후, 규모 확대 시 검토**

## 4. CODEF 없는 MVP 전략 (credit4u PDF 방식)

### 플로우

```
의뢰인이 credit4u.or.kr 접속 (간편인증)
         ↓
개인신용정보 조회서 PDF 다운로드
         ↓
우리 서비스에 PDF 업로드
         ↓
AI가 파싱 (채권자명, 채무원금, 이자, 연체일수 추출)
         ↓
사용자 검증/수정 (테이블 형태로 확인)
         ↓
개인회생 서류 자동 완성
```

### PDF 파싱 기술

| 방법 | 도구 | 정확도 | 비용 | 권장 |
|------|------|--------|------|------|
| 텍스트 추출 | pdf-parse (Node.js), pdfplumber (Python) | 95%+ | 무료 | 1차 시도 |
| Vision API | Claude Haiku/Sonnet | 90-97% | $0.003~0.03/건 | 테이블 복잡할 때 |
| OCR (스크린샷) | Claude Vision, Naver Clova OCR | 85-95% | 유료 | PDF 불가 시 |

### 비용 분석

| 모델 | PDF 1건 비용 | 월 1,000건 | 권장 |
|------|-------------|-----------|------|
| Claude Haiku 3.5 | $0.003~0.01 | **$3~10** | MVP 추천 |
| GPT-4o-mini | $0.001~0.005 | $1~5 | 대안 |
| Claude Sonnet 4 | $0.01~0.03 | $10~30 | 정확도 우선 |

### PDF 방식 vs CODEF 방식 비교

| 항목 | PDF 업로드 + AI | CODEF API |
|------|----------------|-----------|
| 초기 비용 | 거의 없음 | CODEF 계약 필요 |
| 개발 기간 | 2~4주 | 이미 구현됨 |
| UX | 3단계 (외부사이트→다운→업로드) | 1클릭 |
| 데이터 범위 | PDF에 포함된 것만 | 실시간 상세 조회 |
| 인증서 문제 | **없음** (의뢰인이 직접 간편인증) | 타기관 등록 필요 |
| 정확도 | 95%+ (검증 단계 필요) | 100% (정형 JSON) |
| 외부 의존성 | 없음 | CODEF 장애 시 중단 |

## 5. 개인회생 필수 서류 & 데이터 출처 매핑

### 5종 필수 서류

| 서류 | 필요 데이터 | 자동화 소스 | 자동화 수준 |
|------|-----------|-----------|-----------|
| **채권자목록** | 채권자명, 채무원금, 이자, 연체 | credit4u PDF → AI 파싱 | **높음** |
| **재산목록** | 부동산, 차량, 예금, 보험 | payinfo + 공공API | 중간 |
| **수입지출목록** | 월소득, 지출 | 홈택스(CODEF ID/PW) + 수동 | 중간 |
| **신청서** | 인적사항, 채무총액 | 위 데이터 기반 자동생성 | 높음 |
| **변제계획안** | 가용소득, 변제기간 | 자동계산 (공식 기반) | **높음** |

### 활용 가능한 공공 API (data.go.kr)

| API | 용도 | 자동화 |
|-----|------|--------|
| 공동주택 공시가격 | 아파트 재산 평가 | **가능** (REST API) |
| 개별주택/공시지가 | 단독주택/토지 평가 | **가능** (REST API) |
| 실거래가 | 부동산 시세 참고 | **가능** (REST API) |
| 차량 등록정보 | 차량 소유 확인 | **가능** (시세 미포함) |
| 차량 기준가액 | 차량 가치 평가 | 불가 (보험개발원 수동 조회) |
| 등기부등본 | 부동산 권리관계 | 불가 (인터넷등기소 수동) |

## 6. 경쟁 환경 & 시장 기회

### 현재 시장

- CODEF 연동을 통한 개인회생 채무 자동 수집 서비스: **전무 (블루오션)**
- 로톡, 헬프미 등: 변호사 매칭 플랫폼 (서류 자동화 아님)
- 법무사 사무소: 대부분 엑셀 + 수작업
- 개인회생 신청 건수: **연간 10만 건 이상** (증가 추세)

### 해외 사례

- **Upsolve (미국)**: Chapter 7 파산 자동화. 수동 입력 기반 위자드. API 없이 시작.
- **핵심 교훈**: 수동 입력도 UX만 좋으면 충분히 작동. MVP는 수동으로 시작.

### 차별화 포인트

| 기존 서비스 | 우리 서비스 |
|------------|-----------|
| 의뢰인이 서류 직접 수집 (2~4주) | AI가 PDF 파싱으로 자동 추출 |
| 법무사가 엑셀로 정리 | 채권자목록 자동 생성 |
| 변제계획 수동 계산 | 자동 계산 + 최적화 |
| 법원 양식 수동 작성 | DOCX/HWPX 자동 생성 |

## 7. 권장 전략 (Phase별)

### Phase 1: MVP (2~4주) — credit4u PDF + AI 파싱

```
[프론트엔드]
- credit4u PDF 업로드 UI
- AI 파싱 결과 검증/수정 테이블
- 수동 보충 입력 폼 (사채 등)

[백엔드]
- pdf-parse로 텍스트 추출
- Claude Haiku로 JSON 구조화
- 채권자목록 → 법원 양식 자동 생성

[비용]
- AI: 월 $3~10
- 인프라: 기존 Firebase/CF Workers
```

**장점**: CODEF 계약/인증서 문제 없음, 즉시 시작 가능, 비용 최소
**단점**: 의뢰인이 credit4u 방문 필요 (3단계 외부 작업)

### Phase 2: CODEF 간편인증 추가 (Phase 1 이후)

```
- CODEF 간편인증(카카오/PASS)으로 주요 기관 실시간 조회
- 홈택스 ID/PW로 소득 자동 수집
- PDF 방식은 fallback으로 유지
```

### Phase 3: CODEF Credit4U API 하이브리드 (PMF 확인 후)

```
- CODEF → Credit4U 스크래핑으로 전체 채무 일괄 수집
- 1클릭으로 채권자목록 완성
- 공동인증서 서비스(50만원/월) 검토
```
