## 1. 검토 요청 배경

회생클릭(B2C) 출시를 앞두고 **변호사법 위반 리스크, 개인정보보호법/신용정보법 대응, 서비스 경계 설정**에 대한 변호사님의 법률 검토가 필요합니다.

---

## 2. 변호사법 관련 쟁점 (가장 중요)

### 쟁점 1: B2C 셀프 신청 도구의 법적 성격
서비스 개요:
- 채무자 본인이 직접 입력·결제하고 서류를 생성하는 **자동화 도구**
- 김창희 변호사님이 (SELF+ 플랜 한정) 1:1 검수 30분 제공

**검토 요청 사항**:
- [ ] "자동 서류생성 도구" 자체가 **변호사법 제109조(법률사무 유상 수임)** 위반에 해당하지 않는지
- [ ] SELF 플랜(99K, 변호사 비개입)과 SELF+ 플랜(199K, 변호사 검수)의 **법적 경계 문구** 자문
- [ ] 이용약관에 **"본 서비스는 법률자문이 아닙니다"** 명시로 충분한지, 추가 방어 문구 필요 여부

### 쟁점 2: AI 진술서·서류 검증 기능
- Claude API로 "채무 발생 경위" 등 진술문 초안 자동 생성
- 신규 도입: **AI 서류 검증 기능** (제출 전 논리오류·누락 체크)

**검토 요청 사항**:
- [ ] AI가 생성하는 진술문이 **법률자문으로 간주될 소지**가 있는지
- [ ] "초안이며 변호사 확인이 필요합니다" 디스클레이머 수위
- [ ] AI 검증 결과를 "권고"로 표현해야 하는지, "필수 수정"으로 표현 가능한지

### 쟁점 3: 성공보수형 결제 모델 도입 검토
- 아이디어: "인가 받으면 50만원, 못 받으면 0원"
- 현재는 선불제(99K/199K)

**검토 요청 사항**:
- [ ] 변호사 아닌 회사가 **성공보수형 요금제** 운영 시 변호사법 위반 가능성
- [ ] 김창희 변호사님 명의로 운영 vs 회사 명의로 운영 시 차이

## 3. 개인정보보호법 / 신용정보법 쟁점

### 수집 항목
| 항목 | 출처 | 민감도 |
|------|------|--------|
| 주민등록번호 | 이용자 직접 입력 | 🔴 고유식별정보 |
| 은행·카드·보험 계좌 정보 | CODEF API 수집 | 🔴 신용정보 |
| 부동산 등기·공시가격 | 공공데이터 API | 🟠 재산정보 |
| 차량 등록원부 | CODEF API | 🟠 재산정보 |
| 건강보험/국민연금/세금 | CODEF 공공기관 | 🟠 민감정보 |

### 검토 요청 사항
- [ ] **신용정보법 제15조** 적용 대상 여부 — 회사가 "신용정보회사"로 분류될 위험
- [ ] CODEF 이용약관상 "재판매 금지" 조항과 법률 SaaS 운영의 양립 가능성 (CODEF 측에 서면 확인 필요)
- [ ] 주민등록번호 수집 근거 — 개인정보보호법 제24조의2 (대체수단 불가 사유)
- [ ] AES-GCM 암호화 저장은 안전성 확보조치로 충분한지
- [ ] B2C 개인이 "본인 정보"를 입력하는 경우, B2B(법무사가 의뢰인 정보 입력)와 **수집·이용 고지 문구 차이점**

## 4. 서비스 운영상 추가 쟁점

### 4.1 마케팅·광고 표현
- "변호사 수임료의 3%로 변호사 수준의 서류" — 비교광고 표시·광고법 위반 여부
- "인가 확률 X%" 제시 — 결과 보장처럼 인식될 위험
- 실제 인가 사례 콘텐츠화 시 의뢰인 동의 범위

### 4.2 책임 소재
- 서류 오류로 인한 **신청 기각** 시 회사 책임 범위
- CODEF 데이터 오류로 인한 서류 불일치
- 의뢰인의 허위 입력으로 인한 문제 시 면책 가능 수준

### 4.3 광고·제휴 수익
- "추천 변호사 연결 수수료" 모델이 **변호사법 제34조(알선·소개 금지)** 위반 여부
- 김창희 변호사님 외 **협력 변호사 네트워크 구축 시** 법적 구조 (업무위탁 vs 추천)
