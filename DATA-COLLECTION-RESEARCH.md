## 데이터 수집 파이프라인 아키텍처

```
[Phase 1: 크레딧포유 PDF] ← 채무 전체 윤곽 (의뢰인 직접)
  의뢰인 → credit4u.or.kr (간편인증) → PDF 다운로드 → 업로드
  → Cloud Functions → Claude API PDF 파싱 → Firestore 저장
  → 채권자목록 80% 자동 완성

[Phase 2: CODEF 공공기관] ← 인증서 불필요 (ID/PW)
  홈택스 → 소득금액증명, 원천징수영수증
  정부24 → 주민등록등본, 자동차등록원부
  건보공단 → 자격득실확인서, 보험료 납부
  위택스 → 지방세 체납

[Phase 3: CODEF 금융기관] ← 인증서 또는 간편인증
  은행 → 대출 상세, 예금 잔액
  카드사 → 카드론, 청구내역
  보험사 → 보험 계약 (내보험다보여)

[Phase 4: 공공 API] ← API 키만 필요
  부동산 공시가격 (data.go.kr) ← 무료
  자동차 시세 (CODEF used-car API)

[Phase 5: 수동 보완]
  사채, 퇴직금, 임대차계약서, 보험 해약환급금
```

## PDF 파싱 기술 선택

### 권장: Claude API document type (PDF 직접 전송)

```javascript
// Cloud Functions에서 구현
const response = await anthropic.messages.create({
  model: "claude-haiku-4-5-20251001",
  messages: [{
    role: "user",
    content: [
      { type: "document", source: { type: "base64", media_type: "application/pdf", data: pdfBase64 } },
      { type: "text", text: "이 신용조회서에서 채무 정보를 JSON으로 추출..." }
    ]
  }]
});
```

**장점:** PDF 파싱 라이브러리 불필요, 테이블 구조 인식 우수, 포맷 변경에 강건
**비용:** 건당 $0.003~0.03 (Haiku~Sonnet)

### 크레딧포유 PDF 주의사항
- 비밀번호 보호 (주민번호 앞6자리) → 복호화 필요
- 텍스트 기반 PDF (스캔 아님) → 파싱 정확도 높음

## 의뢰인 데이터 수집 UX

### 최소 작업 플로우 (의뢰인 기준)

```
[1단계] 기본정보 입력 (카톡으로 가능) — 3분
  이름, 생년월일, 연락처

[2단계] 크레딧포유 PDF 다운 — 5분 ★핵심
  → 간편인증(카카오)으로 로그인
  → 신용조회서 PDF 다운로드
  → 카톡/앱으로 전송
  → AI가 채권자목록 80% 자동 생성

[3단계] 추가 서류 (선택) — 10분
  → 급여명세서, 임대차계약서 사진 촬영
  → 앱에서 업로드

[4단계] 검증 — 5분
  → AI가 추출한 결과 확인/수정
```

### 인증서 없는 의뢰인 대응
1. **간편인증(카카오/PASS)** 우선 안내
2. **금융인증서** (은행 앱에서 즉시 발급)
3. **은행 영업점 방문** 안내 (최후 수단)

## 자동화 가능 범위 요약

| 서류 | 자동화 비율 | 수동 필요 항목 |
|------|:---------:|-------------|
| 채권자목록 | **70~80%** | 사채, 대부업, 채권양도 확인, 정확한 이자 |
| 재산목록 | **60~70%** | 보험 해약환급금, 퇴직금, 임차보증금 |
| 수입지출목록 | **50~60%** | 현금소득, 추가인정 생계비 항목 |
| 신청서 | **90%** | 사건 경위 진술 |
| 변제계획안 | **95%** | 법무사 최종 검토 |

**기존 수작업 소요:** 2~4주 → **자동화 후:** 1~2일
